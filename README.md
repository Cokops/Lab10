## Описание работы

В данной лабораторной работе реализовано взаимодействие между веб-сервисами на Python  и Go  для решения задач построения API, обмена сложными JSON-структурами, организации корректного завершения работы сервисов, а также реализовано gRPC-взаимодействие и контейнеризация через Docker Compose.

## Выполненные задания

### Средней сложности

**Задание 1: Создать простое API на Go (Gin) с 2-3 эндпоинтами (Задание 1 по Методичке)**
*   Реализован веб-сервер на Go с использованием фреймворка Gin (файл `ZadMid/go-service/main.go`).
*   Созданы эндпоинты:
    *   `GET /health` — проверка состояния сервиса
    *   `GET /data` — получение списка данных в формате JSON
    *   `POST /data` — добавление новых данных
*   Добавлено middleware для логирования всех входящих запросов.
*   Реализована валидация входных данных для `POST /data` (проверка наличия полей `id`, `name`, `value`).

**Задание 5: Передавать сложные структуры данных (JSON) между сервисами (Задание 5 по Методичке)**
*   Реализована передача сложных JSON-структур между Go и Python сервисами.
*   Структуры включают:
    *   Вложенные объекты (`metadata` с полями `created_at`, `author`, `version`)
    *   Массивы (`tags` для категоризации данных)
    *   Разнородные типы данных (строки, числа, булевы значения)
*   Python-сервис при запросе `GET /data` самостоятельно обращается к Go-сервису, получает данные и объединяет их с собственными.

**Задание 7: Реализовать graceful shutdown в обоих сервисах (Задание 7 по Методичке)**
*   В обоих сервисах реализована корректная обработка сигналов завершения (`SIGINT`, `SIGTERM`):
    *   **Go:** `signal.Notify` + graceful завершение HTTP-сервера с ожиданием завершения текущих запросов
    *   **Python:** `signal.signal` + обработчик для корректного завершения воркеров
*   При получении сигнала сервисы завершают текущие запросы, закрывают соединения и выходят без потери данных.

### Повышенной сложности

**Задание 1: Реализовать gRPC-сервер на Go и gRPC-клиент на Python (Задание 1 по Методичке)**
*   Реализовано gRPC-взаимодействие между сервисами:
    *   **gRPC-сервер на Go:**
        *   Определён protobuf-контракт (`proto/data.proto`) с методами `GetData` и `AddData`
        *   Сгенерирован код через `protoc` с использованием `protoc-gen-go-grpc`
        *   Запущен на порту `50051` параллельно с REST API
    *   **gRPC-клиент на Python:**
        *   Сгенерированы классы из `.proto` файла с использованием `grpcio-tools`
        *   Реализована асинхронная отправка запросов к Go-серверу
        *   Выполняет проверку связи и получение данных через gRPC

**Задание 5: Развернуть оба сервиса в Docker Compose с общей сетью (Задание 5 по Методичке)**
*   Создан `docker-compose.yml`, объединяющий все компоненты:
    *   **go-service:** Gin REST API + gRPC сервер (порты `8080`, `50051`)
    *   **python-service:** Flask REST API + gRPC клиент (порт `5000`)
    *   Общая сеть `backend` для взаимодействия через Docker DNS
    *   Healthcheck для обоих сервисов (проверка `/health` каждые 30 секунд)
    *   Зависимости и порядок запуска через `depends_on`
*   Созданы `Dockerfile` для обоих сервисов:
    *   **Go:** многоэтапная сборка на основе `golang:1.21-alpine`
    *   **Python:** сборка на основе `python:3.11-slim`
